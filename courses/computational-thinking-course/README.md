# computational-thinking-course

## Overview

**computational-thinking-course** is an introductory course focused on the foundations of computational thinking before students begin coding in a specific programming language.

This course develops the ability to **analyze problems**, **break them into smaller parts**, **design step-by-step solutions**, and **represent those solutions clearly** through algorithms, flowcharts, and pseudocode. It also introduces the basic concepts of programming, programming languages, computer components, and programming paradigms.

The course is designed to serve as the conceptual bridge between **problem solving** and the later implementation of solutions in a language-specific course such as **cs-course**.

---

## Course Objective

Build a strong foundation in computational thinking so students can understand how to approach problems logically, model solutions, and prepare for programming in C# and other languages.

---

## Learning Outcomes

By the end of this course, students will be able to:

- Understand problem solving as a structured and repeatable process.
- Identify, analyze, and break down real-world problems into manageable parts.
- Design algorithms as step-by-step solutions.
- Represent algorithms using flowcharts and pseudocode.
- Understand what programming is and how it relates to problem solving.
- Recognize the role of programming languages, compilers, and computer hardware.
- Differentiate between types of programming languages and major programming paradigms.

## Course Structure

## Module 01 - Problem Solving and Algorithm Design

This module introduces the mindset required before writing code. Students learn that programming is ultimately about solving problems and that every good solution begins with a clear understanding of the problem itself.

### 1. Problem Solving

Before building software, it is necessary to understand the problem that needs to be solved.

**Definition:**
Problem solving is the process of identifying and resolving challenges or obstacles in a logical and effective way.

### Problem Solving Steps

1. **Identify and define the problem**
   - Understand the problem clearly.
   - Break it into smaller and manageable parts.
   - Identify the main cause of the problem.

2. **Gather information**
   - Research the problem.
   - Collect data.
   - Talk to people involved or familiar with the situation.

3. **Analyze the problem**
   - Examine the collected information.
   - Detect patterns, constraints, and possible causes.
   - Use tools such as cause-and-effect diagrams or SWOT analysis when helpful.

4. **Develop possible solutions**
   - Propose multiple alternatives.
   - Brainstorm ideas.
   - Consider creative and structured approaches.

5. **Evaluate possible solutions**
   - Compare pros and cons.
   - Consider feasibility, cost, and impact.
   - Select the most promising option.

6. **Implement the solution**
   - Create an action plan.
   - Assign responsibilities.
   - Define a timeline.
   - In programming, this eventually becomes the development of an application.

7. **Monitor and adjust**
   - Review whether the solution works.
   - Detect errors or issues.
   - Gather feedback.
   - Adjust the solution when necessary.

### Example Contexts

Students apply these steps to everyday situations such as:

- Organizing a group trip with different budgets and preferences.
- Diagnosing why a computer has become slow.
- Deciding food options for a party.
- Fixing a bicycle chain.
- Packing efficiently for a trip.

---

### 2. Algorithms

Once a problem is understood, the next step is to define a sequence of actions to solve it.

**Definition:**
An algorithm is a process that describes step by step how to solve a problem.

In other words, it is a set of instructions that guides the completion of a task.

Algorithms can be compared to recipes: just as a recipe explains what ingredients are needed and which steps must be followed, an algorithm explains what actions must be performed to reach a solution.

### Everyday Examples of Algorithms

- Preparing a sandwich.
- Washing clothes.
- Making tea.
- Converting Fahrenheit to Celsius.
- Searching for an element in a list.

### Algorithms in Programming

In computer science, algorithms define the sequence of steps that will later be translated into code. For example, a navigation application may use an algorithm to:

- Identify the starting point and destination.
- Calculate possible routes.
- Select the shortest route.

---

### 3. Flowcharts

Flowcharts are the first formal way to represent an algorithm visually.

**Definition:**
Flowcharts use symbols and simple shapes to represent actions, decisions, inputs, and outputs in a process.

### Core Flowchart Elements

- **Start/End**: indicates the beginning or end of a process.
- **Process/Task**: represents an action to perform.
- **Input/Output**: represents data entering or leaving the process.
- **Decision**: represents a point where a condition determines the next path.
- **Connector**: connects separate sections of a large diagram.
- **Arrow**: shows the direction and order of the process.

### Why Flowcharts Matter

Flowcharts help students:

- Visualize a process clearly.
- Understand the order of steps.
- Identify decisions and branches.
- Simplify complex tasks.

### Suggested Practice Examples

- Going to the store to buy a pen.
- Calculating the average of two numbers.
- Printing the numbers from 1 to 5.

### Tools

Some online tools that can be used to build flowcharts include:

- diagrams.net
- Lucidchart

A flowchart cheat sheet can also be included as supporting material for future projects.

---

### 4. Pseudocode

Pseudocode is the second major way to represent algorithms.

**Definition:**
Pseudocode is a structured but language-independent way of describing the logic of an algorithm.

It is not a real programming language. Instead, it helps students plan and design a solution before writing the final code.

### Why Pseudocode Matters

Pseudocode helps students:

- Focus on logic before syntax.
- Describe algorithms in a clear, readable format.
- Discuss solutions without depending on a specific programming language.
- Detect mistakes before coding.

### Typical Pseudocode Actions

- `INICIO / FIN`
- `LEER`
- `ESCRIBIR`
- variable assignment
- arithmetic operations
- conditions such as `SI / SINO`
- loops such as `MIENTRAS`

### Suggested Practice Examples

- Sum of two numbers.
- Printing numbers from 1 to 5.
- Fahrenheit to Celsius conversion.
- Logging in to a social network.

### Module 01 Closing Idea

By combining **problem solving**, **algorithms**, **flowcharts**, and **pseudocode**, students begin to think like programmers before writing actual code.

## Module 02 - Programming Foundations

After students understand how to analyze problems and represent solutions, they move into the conceptual foundations of programming.

### 1. Programming

**Definition:**
Programming is the process of creating software by writing instructions in a special language that computers can understand.

Programming can be understood as giving precise instructions to a computer so it can perform a task step by step.

The process usually involves:

- Thinking about what the computer should do.
- Breaking a large problem into smaller parts.
- Designing the logic of the solution.
- Writing the solution in a programming language.
- Executing and testing the program.
- Debugging errors if necessary.

This connects directly with the ideas developed in Module 01.

---

### 2. Programming Languages

**Definition:**
Programming languages are formal systems of instructions and rules that allow humans to communicate with computers.

Computers ultimately process binary code (`0` and `1`), so programming languages serve as a bridge between human-readable logic and machine execution.

### Examples of Programming Languages

- Python
- JavaScript
- Java
- C#
- C / C++

These languages can be used to build websites, mobile apps, desktop applications, games, and many other kinds of software.

---

### 3. Computer Components

To understand programming better, students are also introduced to the main components of a computer system.

#### Input Units
Devices used to provide information to the computer.

Examples:
- Keyboard
- Mouse
- Touchscreen
- Microphone
- Camera
- USB devices
- Barcode readers

#### Output Units
Devices used to present processed information to the user.

Examples:
- Monitor
- Printer
- Headphones
- Multimedia devices
- VR / mixed reality headsets
- Storage devices when receiving data for saving

#### Memory Unit (RAM)
Stores data temporarily so it can be accessed and processed quickly.

#### Arithmetic Logic Unit (ALU)
Performs arithmetic and logical operations such as:
- addition
- subtraction
- multiplication
- division
- comparisons and decision-related processing

#### Central Processing Unit (CPU)
Coordinates and supervises the work of the other computer units.

#### Graphics Processing Unit (GPU)
Handles graphics rendering and highly parallel computations, including modern applications in AI and data processing.

## Module 02 - Types and Paradigms of Programming Languages

This section completes the conceptual introduction to programming.

### 1. Types of Programming Languages

Programming languages can be grouped into two major categories:

#### High-Level Languages
These languages are easier for humans to read and write.

- Closer to natural language.
- Require translation before execution.
- Common examples: Python, JavaScript, Java.

#### Low-Level Languages
These languages are closer to machine instructions.

- Faster and more direct.
- Give more control over hardware.
- Usually harder to learn and use.
- Common examples: Assembly and C.

---

### 2. Compilers

A compiler works like a translator between the programmer and the computer.

**Compiler responsibilities:**

1. Read the source code.
2. Check its structure and grammar.
3. Translate it into a form the computer can execute.
4. Optimize the translated code when possible.
5. Produce executable code.

This explains why programmers usually work in high-level languages instead of writing machine code directly.

---

### 3. Programming Paradigms

Programming paradigms are ways of structuring and thinking about programs.

#### Imperative Programming
A program is written as a sequence of step-by-step instructions that explicitly control the computer.

#### Procedural Programming
A branch of imperative programming that organizes code into functions or procedures.

#### Declarative Programming
Focuses on describing **what result is desired**, rather than specifying every step of **how** to get it.

#### Object-Oriented Programming
Organizes programs around objects that contain properties and behaviors.

#### Event-Driven Programming
Builds applications that react to events such as button clicks, key presses, or mouse actions.

#### Visual Programming
Uses graphical elements such as blocks instead of traditional typed code.
